# План двухнедельной практики

## Общая схема

```
Неделя 1: Погружение → Архитектура → Простые утилиты
Неделя 2: Сложные утилиты → Тесты → Защита
```

## Ожидаемый результат

По итогам двух недель каждый студент должен:

- Реализовать **2-3 утилиты** (минимум одна сложности ★★ или ★★★).
- Пройти полный цикл: backend → frontend → тесты → GitHub PR → code review → merge.
- Развернуть приложение в Docker.
- Продемонстрировать работу на защите.
